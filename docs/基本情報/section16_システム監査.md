# 【Section 16】システム監査

## 1. システム監査の目的と役割
独立した客観的な立場にある第三者（システム監査人）が、情報システムのリスクに対するコントロール（内部統制）が適切に整備・運用されているかを評価し、改善のための助言を行う活動です。
* システム監査人は、自らシステムの構築や運用に関わってはならず（独立性の原則）、客観的な監査証拠に基づいて判断を下します。

## 2. コーポレートガバナンスと内部統制
* **コーポレートガバナンス (企業統治)**: 企業が健全で効率的な経営を行うための管理体制。
* **内部統制**: 企業の事業目的（業務の有効性・効率性、財務報告の信頼性、法令遵守、資産の保全）を達成するために、企業内部で構築・運用されるルールや仕組みのこと。
* **ITガバナンス**: 経営陣が、IT戦略を経営戦略と一致させ、IT投資の価値を最大化し、ITリスクを最適化するための組織的な能力。
