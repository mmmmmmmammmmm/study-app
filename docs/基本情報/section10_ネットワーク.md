# 【Section 10】ネットワーク

## 1. ネットワーク階層モデル (OSI基本参照モデル)
異なるメーカーのコンピュータ同士が通信できるように、通信の機能を7つの階層（レイヤ）に分割して標準化したモデルです。
1.  **物理層**: ケーブルの材質や電気信号の仕様（機器：リピータ、ハブ）。
2.  **データリンク層**: 同一ネットワーク内での通信。MACアドレスを用いて隣接機器へデータを送る（機器：L2スイッチ）。
3.  **ネットワーク層**: 異なるネットワーク間の通信。IPアドレスを用いて最適な経路を選択（ルーティング）する（機器：ルータ、L3スイッチ）。
4.  **トランスポート層**: 通信の信頼性を確保する。エラー時の再送制御など（プロトコル：TCP、UDP）。
5.  **セッション層**: アプリケーション間の論理的な通信路（セッション）の確立と解放。
6.  **プレゼンテーション層**: データの表現形式（文字コード、暗号化、圧縮）の変換。
7.  **アプリケーション層**: ユーザーが利用する具体的なネットワークサービス（Web、メール等）を提供する。

## 2. TCP/IPと主要なプロトコル
現在のインターネットで事実上の標準となっているプロトコル群です。
*   **IP (Internet Protocol)**: インターネット上の住所である「IPアドレス」を用いてパケットを配送します。IPv4（32ビット）が枯渇したため、現在はIPv6（128ビット）への移行が進んでいます。NAT/NAPTは、社内の「プライベートIPアドレス」とインターネット上の「グローバルIPアドレス」を変換する技術です。
*   **TCP と UDP**:
    *   **TCP**: 通信相手とコネクションを確立し、データが届いたか確認（ACK）しながら送る「高信頼・低速」なプロトコル。Web(HTTP)やメールで使われます。
    *   **UDP**: 到達確認を行わず、データを送りっぱなしにする「低信頼・高速・低遅延」なプロトコル。動画ストリーミング、音声通話(VoIP)、DNSで使われます。
*   **DHCP**: ネットワークに接続したPCに対し、IPアドレスなどの設定情報を「自動的」に割り当てるプロトコルです。
*   **DNS**: 人間が覚えやすい「ドメイン名（例: www.example.com）」と、コンピュータが処理する「IPアドレス」を相互に変換（名前解決）するプロトコルです。
