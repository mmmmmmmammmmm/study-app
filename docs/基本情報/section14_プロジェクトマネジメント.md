# 【Section 14】プロジェクトマネジメント

## 1. プロジェクトマネジメント (PMBOK)
プロジェクト（独自の目的を、期限を設けて達成するための活動）を管理する世界標準の体系です。

## 2. 主要な管理領域
* **スコープマネジメント**: プロジェクトで「やるべき作業」と「やらない作業」を定義します。**WBS (Work Breakdown Structure)** を作成し、作業を階層的に細分化・構造化します。
* **スケジュール（時間）マネジメント**: 作業の順序関係を**アローダイアグラム（PERT図）**で可視化します。プロジェクト全体の最短所要時間を決定する、余裕時間ゼロの経路を**クリティカルパス**と呼びます。
* **コストマネジメント**: 予算の算出と統制を行います。EVM（アーンド・バリュー・マネジメント）を用いて、計画値(PV)、出来高(EV)、実績コスト(AC)から進捗とコストの状況を定量的に分析します。
* **品質マネジメント**: パレート図（重要な問題を特定する）、特性要因図（フィッシュボーン・チャート：原因と結果の関係を整理する）、管理図（異常の兆候を検知する）などのQC7つ道具を用いて品質を管理します。
