# 【Section 12】システム開発技術

## 1. ソフトウェア開発のライフサイクル
システム開発は一般的に、以下の工程（フェーズ）を順を追って進めます。
1.  **システム要件定義**: ユーザーへのヒアリングを通じて、システムが「何をしなければならないか（要件）」を明確にします。「業務要件」「機能要件（必要な機能）」「非機能要件（性能、セキュリティ、可用性など）」を定義します。
2.  **設計**: 要件定義をもとに、システムをどのように作るかを決定します。
    *   **外部設計（基本設計）**: ユーザーから見える部分（画面レイアウト、帳票フォーマット、操作手順など）の設計。
    *   **内部設計（詳細設計）**: 開発者向けに、システムの内部処理、データベースの物理構造、モジュール（部品）の分割方法などを設計。
3.  **実装（プログラミング）**: 詳細設計書に従い、ソースコードを記述します。
4.  **統合・テスト**: 作成したプログラムが要件通りに正しく動作するかを検証します。
5.  **導入・受入れ支援**: 完成したシステムを本番環境に展開し、ユーザー部門が実際に業務で使えるかを確認（受入テスト）します。

## 2. テスト手法
バグ（欠陥）を早期に発見するため、小さな単位から徐々に大きな単位へと段階的にテストを行います。
*   **単体テスト**: プログラム（モジュール）単体が正しく動作するかを検証します。主に開発者が、プログラムの内部構造を網羅的に確認する「ホワイトボックステスト」を実施します。
*   **結合テスト**: 複数のモジュールを繋ぎ合わせ、モジュール間のデータの受け渡し（インターフェース）が正しく行われるかを検証します。上位モジュールの代わりをする「ドライバ」や、下位モジュールの代わりをする「スタブ」を用いて効率的にテストを進めます。
*   **システムテスト（総合テスト）**: システム全体が完成した状態で、要件定義書で定められた機能や性能（非機能要件）を満たしているかを検証します。プログラムの内部構造は見ず、入力に対する出力結果だけを確認する「ブラックボックステスト」が中心となります。
