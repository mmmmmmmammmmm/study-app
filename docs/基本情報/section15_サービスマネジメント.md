# 【Section 15】サービスマネジメント

## 1. ITサービスマネジメントとITIL
情報システムを単なる「システム」としてではなく、ユーザーに対する「サービス」として提供・管理するためのベストプラクティス集が**ITIL**です。

## 2. 主要なプロセス
* **インシデント管理**: システム障害やユーザーの問い合わせ（インシデント）が発生した際、**可能な限り迅速にサービスを復旧させる**（回避策の提供など）ことを目的とします。
* **問題管理**: インシデントの**根本原因を突き止め、恒久的な解決策を見出す**プロセスです。再発防止を目的とします。
* **変更管理**: システムへの変更（パッチ適用、設定変更など）が引き起こすリスクを評価し、変更を許可するか却下するかを決定します。
* **SLA (Service Level Agreement)**: サービスの提供者と利用者の間で合意する「サービス品質の保証基準（稼働率99.9%など）」の取り決め。合意したレベルを維持・改善する活動をSLM（サービスレベルマネジメント）と呼びます。
