# 【Section 4】システム構成要素

## 1. システム構成方式
システムを止めない（高可用性）ための様々な構成方式があります。
*   **デュアルシステム**: 2つのシステムで全く同じ処理を独立に行い、結果を照合（クロスチェック）します。一方が故障しても処理を継続できる、極めて信頼性の高い構成です。
*   **デュプレックスシステム**: 「稼働系（現用系）」と「待機系（予備系）」を用意する構成。障害発生時に待機系に切り替えます。常に電源を入れ同期しておく「ホットスタンバイ」と、障害が起きてから立ち上げる「コールドスタンバイ」があります。
*   **RAID (Redundant Arrays of Inexpensive Disks)**: 複数のハードディスクを仮想的に1つのドライブとして扱う技術。
    *   RAID1（ミラーリング）：2台のディスクに全く同じデータを書き込み、冗長性を持たせます。
    *   RAID5（パリティ分散）：データと誤り訂正用のパリティを3台以上のディスクに分散して書き込み、高速化と耐障害性を両立します。

## 2. システムの評価指標 (RASIS)
*   **Reliability (信頼性)**: 故障のしにくさ。MTBF（平均故障間隔：正常に稼働している時間の平均）で評価します。
*   **Availability (可用性)**: 必要な時にシステムが使える割合。「稼働率 ＝ MTBF / (MTBF ＋ MTTR)」で計算します。
*   **Serviceability (保守性)**: 故障時の直しやすさ。MTTR（平均修復時間）で評価します。
