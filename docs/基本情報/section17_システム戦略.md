# 【Section 17】システム戦略

## 1. 情報システム戦略
経営戦略に基づいて、企業全体における情報システムの構築・運用の基本方針を定めることです。
* **エンタープライズアーキテクチャ (EA)**: 組織全体の業務とシステムを統一的な手法でモデル化し、全体最適化を図るフレームワーク。以下の4つの階層（アーキテクチャ）で構成されます。
    1. ビジネス・アーキテクチャ (業務)
    2. データ・アーキテクチャ (データ構造)
    3. アプリケーション・アーキテクチャ (処理)
    4. テクノロジ・アーキテクチャ (基盤技術)

## 2. 業務プロセスの改善
* **BPR (Business Process Reengineering)**: 既存の業務プロセスを抜本的に見直し、再構築すること。
* **BPM (Business Process Management)**: 業務プロセスを継続的に測定・評価・改善していく管理手法（PDCAサイクルを回す）。
* **RPA (Robotic Process Automation)**: デスクトップ上の定型的な事務作業（データ入力など）を、ソフトウェアロボットに代行させる技術。
