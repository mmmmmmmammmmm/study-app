# 【Section 5】ソフトウェア

## 1. オペレーティングシステム (OS)
ハードウェアを隠蔽し、アプリケーションソフトウェアに共通のインターフェースを提供する基本ソフトウェア（Windows, Linuxなど）です。
*   **タスク管理 (プロセス管理)**: 実行中のプログラム単位をタスクと呼びます。OSはタスクを「実行可能状態」「実行状態」「待機状態」の3つで遷移させます。CPUの使用権を割り当てることを「ディスパッチ」、強制的に奪うことを「プリエンプション」と呼びます。
*   **記憶管理 (仮想記憶)**: 物理メモリ（主記憶）の容量不足を補うため、補助記憶（HDDなど）の一部をあたかも主記憶の延長のように見せかける技術です。
    *   **ページング方式**: プログラムとメモリを固定長の「ページ」に分割して管理します。必要なページがメモリ上にない場合「ページフォールト」が発生し、メモリと補助記憶間でページを入れ替える「スワッピング」が行われます。スワッピングが頻発しシステムが極端に遅くなる現象を「スラッシング」と呼びます。

## 2. 開発ツールとオープンソース
*   **言語プロセッサ**: 人間が書いたソースコードを機械語に変換する「コンパイラ」や、複数のモジュールを結合して実行可能ファイルを作る「リンカ（連係編集プログラム）」があります。
*   **オープンソースソフトウェア (OSS)**: ソースコードが公開され、誰でも自由に使用・改変・再配布できるソフトウェア。GPL（コピーレフト：改変物も公開義務があるライセンス）やApacheライセンスなどがあります。
