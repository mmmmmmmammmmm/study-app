# 【Section 13】ソフトウェア開発管理技術

## 1. 開発モデル（手法）
* **ウォーターフォールモデル**: 上流工程から下流工程へと滝の水が落ちるように順番に進める手法。後戻り（手戻り）が難しいが、進捗管理がしやすい。大規模開発向け。
* **アジャイル開発**: 短期間（イテレーション / スプリント）で小さな機能単位での「設計・実装・テスト」を繰り返し、徐々にシステムを完成させていく手法。仕様変更に強く、ユーザーのフィードバックを早期に取り入れられる。代表的な手法に「スクラム」がある。
* **スパイラルモデル**: システムを独立性の高い部分（サブシステム）に分割し、ユーザーの確認を得ながら設計・プログラミング・テストを繰り返して完成度を高める。

## 2. 構成管理・変更管理
* **構成管理**: ソースコードや設計書のバージョン（履歴）を管理し、誰がいつ変更したか、過去のバージョンに戻せるようにする仕組み（Gitなど）。
* **変更管理**: ユーザーからの仕様変更要求に対して、影響範囲やコスト、スケジュールへの影響を評価し、承認されたものだけを計画に組み込むプロセス。
