# 【Section 18】システム企画

## 1. システム化計画と要件定義
システム開発の最初に行う、プロジェクトの方向性を決めるフェーズです。
* **システム化計画**: 経営課題を解決するために、どの業務をシステム化するか、費用対効果（ROI）はどうか、スケジュールはどうするかを計画します。
* **要件定義**: システムが備えるべき機能や性能を明確にします。
    * **機能要件**: 「顧客を検索できる」「帳票を出力できる」など、システムが必ず実行しなければならない具体的な機能。
    * **非機能要件**: 可用性（稼働率）、パフォーマンス（応答時間）、セキュリティ、拡張性など、機能以外の品質に関する要件。

## 2. 調達計画・実施
システム開発を外部のベンダー（SIer）に委託するための手続きです。
* **RFI (Request For Information: 情報提供依頼書)**: ベンダーに対して、企業の概要や技術動向、実績などの情報提供を求める文書。
* **RFP (Request For Proposal: 提案依頼書)**: 発注先候補のベンダーに対して、システムの要件や調達条件を提示し、具体的なシステム構成、スケジュール、見積額などの提案を求める文書。
